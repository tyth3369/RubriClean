# 技术方案说明

## 1. 方案概述

采用 **HSV + RGB 混合方法（方案 C）**检测红笔痕迹，使用 OpenCV Inpainting 进行图像修复。

## 2. 算法 Pipeline

```
输入图片
    │
    ├─→ 预处理模块 ──→ 去黑边、对比度归一化
    │
    ├─→ HSV 检测模块 ──→ 掩码 A（标准红色 0°~10° 和 170°~180°）
    │
    ├─→ RGB 差值检测模块 ──→ 掩码 B（R - max(G,B) > 阈值，捕获偏黑红笔）
    │
    ├─→ 黑边排除模块 ──→ 掩码 C（排除图片边缘黑色区域）
    │
    └─→ 掩码融合 ──→ (A ∪ B) - C
              │
              ▼
        形态学优化（膨胀 + 腐蚀，连接断裂笔画、去除噪声）
              │
              ▼
        Inpainting 修复（Telea 或 Navier-Stokes 算法）
              │
              ▼
        输出图片
```

## 3. 模块详述

### 3.1 预处理模块
- **去黑边**：检测边缘连续黑色像素带，裁剪或标记排除
- **对比度调整**：可选 CLAHE（自适应直方图均衡化），增强红色与黑色的区分度

### 3.2 HSV 红色检测
OpenCV 中 HSV 范围：
- H (Hue): 0-180（OpenCV 将 360° 映射到 0-180）
- S (Saturation): 0-255
- V (Value): 0-255

红色在 Hue 环的两端：
- **下区间**：H ∈ [0, 10]，S ∈ [50, 255]，V ∈ [50, 255]
- **上区间**：H ∈ [170, 180]，S ∈ [50, 255]，V ∈ [50, 255]

### 3.3 RGB 差值检测
对偏黑红笔，利用红色墨迹在 R 通道强、G/B 通道弱的特点：
- 计算 `diff = R - max(G, B)`
- 阈值 `diff > 15`（可调）标记为候选红笔像素
- 辅助条件：`R > 80`（排除纯黑像素误检）

### 3.4 掩码融合与优化
- 两个掩码取逻辑或（并集）
- 形态学闭运算（先膨胀后腐蚀）：连接断裂的笔画
- 形态学开运算（先腐蚀后膨胀）：去除孤立噪声点
- 核大小：3×3 或 5×5（取决于图片分辨率）

### 3.5 Inpainting 修复
- **Telea 算法** (`cv2.INPAINT_TELEA`)：速度快，适合细笔画
- **Navier-Stokes 算法** (`cv2.INPAINT_NS`)：质量更高，适合粗笔画
- 修复半径：3-5 像素（匹配笔迹粗细）

## 4. 关键参数

| 参数 | 默认值 | 范围 | 说明 |
|------|:------:|:----:|------|
| `hsv_lower_red1` | [0, 50, 50] | — | 红色下区间下限 |
| `hsv_upper_red1` | [10, 255, 255] | — | 红色下区间上限 |
| `hsv_lower_red2` | [170, 50, 50] | — | 红色上区间下限 |
| `hsv_upper_red2` | [180, 255, 255] | — | 红色上区间上限 |
| `rgb_diff_threshold` | 15 | 5-40 | RGB 差值阈值 |
| `rgb_r_min` | 80 | 50-120 | R 通道最小值（排除纯黑） |
| `morph_kernel_size` | 3 | 1-7 | 形态学核大小 |
| `inpaint_radius` | 3 | 1-10 | Inpainting 半径 |

> **注意**：上述参数为初始默认值，需在样例图片上调试后确定最终值。

| 库 | 版本 | 用途 |

## 6. 文件输出规范

- 单图处理：输入 `name.jpg` → 输出 `name_clean.jpg`
- 批量处理：保持原目录结构，在指定输出目录中镜像生成
- 可选：同时输出红笔掩码可视化图片（调试用）
