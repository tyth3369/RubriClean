# v3 改进项测试计划

> 目标：在不引入深度学习的前提下，测试 3 个有望提升 v3 的改进项

---

## 改进项总览

| 编号 | 改进项 | 来源 | 解决的问题 | 风险 |
|:--:|------|------|------|:--:|
| A | CLAHE 预处理 | 文献 + 工程经验 | 不同页扫描亮度差异导致同一阈值效果不一致 | 低 |

## 改进 A：CLAHE 预处理

### 原理
当前 v3 对所有页用同一组全局阈值。但 116 页中亮度差异很大（227-246）。CLAHE（Contrast Limited Adaptive Histogram Equalization）在 LAB 空间对 L 通道做局部对比度增强，让暗页变亮、亮页变暗，统一到相似亮度水平后再做颜色检测。

### 实现
在 `detect_red_pen()` 最开始加入：
```python
lab = cv2.cvtColor(image, cv2.COLOR_BGR2LAB)
l, a, b = cv2.split(lab)
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
l = clahe.apply(l)
lab = cv2.merge([l, a, b])
image = cv2.cvtColor(lab, cv2.COLOR_LAB2BGR)
```

### 验证
在 p010（亮）、p040（中）、p100（中暗）上跑 v3 + CLAHE，看掩码像素数变化和是否有新增误检。

## 改进 B：DBSCAN 自适应去噪

### 原理
当前 v3 用固定核（3×3 开运算）去噪——这会把孤立的噪声点去掉，但也可能把细笔画中的小片段误删。DBSCAN 按空间密度聚类：真正的红笔笔画是密集的点簇，孤立的噪声点是稀疏的。DBSCAN 可以更精准区分。

### 实现
在 `merge_masks()` 的形态学步骤之前：
1. 提取掩码中所有白色像素的坐标
2. 用 DBSCAN(eps=5, min_samples=10) 聚类
3. 标记为噪声（label=-1）的小簇移除
4. 保留大簇

### 验证
对比 DBSCAN 去噪和当前开闭运算的结果，看哪种保留更多细笔画、去除更多噪声。

## 改进 C：自适应膨胀核

### 原理
当前膨胀核固定为 5×5 椭圆。但对于粗笔画（对勾，可能 8-15 像素宽）和细笔画（红笔文字，可能 3-5 像素宽），同一个核对前者膨胀不够、对后者可能过度。

### 实现
在 `expand_along_strokes()` 中：
1. 对掩码做连通域分析
2. 估算每个连通域的笔画宽度（距离变换）
3. 按宽度分档：细笔画用小核(3×3)、粗笔画用大核(7×7)

### 验证
在 p100（粗对勾）和 p080（细文字）上对比固定核和自适应核。

## 实施顺序

1. **改进 A (CLAHE)** — 最低风险，单独测试
2. **改进 B (DBSCAN)** — 中风险，在 A 基础上测试
