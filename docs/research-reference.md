## 一、相关论文

### 1. DeepErase：弱监督文档墨迹伪影清除（WACV 2020）⭐

| 项目 | 内容 |
|------|------|
| 作者 | W. Ronny Huang 等 (Ernst & Young) |
| 论文 | [arxiv.org/abs/1910.07070](https://arxiv.org/abs/1910.07070) |
| 代码 | [github.com/yikeqicn/DeepErase](https://github.com/yikeqicn/DeepErase) |

**核心思路**：用 U-Net 做二值语义分割，识别文档中的"墨迹伪影"（下划线、涂抹、飞白笔画等），生成伪影掩码后移除。

**与我们项目的关联**：
- 方法可直接迁移：把"墨迹伪影"换成"红笔痕迹"
- 弱监督训练方式可用：自动合成带红笔痕迹的训练数据，无需人工标注
- 实验证明：OCR 准确率提升 40-60%

**训练数据生成方式**：从干净文档出发，程序化叠加真实红笔痕迹（从扫描中裁剪），叠加时自动获得像素级标注。这对我们很有参考价值——可以从未批改的干净作业出发，叠加 116 页中提取的真实红笔笔画。

### 2. 试卷手写答案自动擦除（CSDN 开源项目）⭐⭐⭐

| 项目 | 内容 |
|------|------|
| 来源 | [CSDN blog: zeta9](https://blog.csdn.net/zeta9/article/details/161532379) |
| 模型 | BiSeNetV2 + NAFA（非局部注意力融合） |

**架构设计**：

| 模块 | 作用 |
|------|------|
| BiSeNetV2 | 结构锚定 — 保留印刷文字边缘、网格线、版面布局 |
| NAFA | 纹理拼接 — 用自注意力机制无缝重建纸张背景纹理 |

**为什么不用 GAN**：GAN 虽然 PSNR 高 1.2dB，但会"幻觉"出虚假细节，导致 OCR 准确率反而下降 7.3%。

**训练策略**：
- 阶段一：Dice Loss + L1 Loss → 优化掩码定位 (IoU)
- 阶段二：纯 L1 Loss → 精细化像素级背景重建

**推理优化**：512×512 滑动窗口 + 镜像填充 + 中心裁剪融合，将边缘伪影率从 18.3% 降至 1.9%。

**真实部署数据**（某中学月考）：

| 学校类型 | 日均处理量 | 单页耗时 | OCR 准确率提升 |
|---------|:--------:|:------:|:-----------:|
| 重点中学 | 12,400 张 | 38.2s (RTX 3060) | 61.3% → 92.7% |
| 普通中学 | 8,900 张 | 52.7s (GTX 1660) | 54.8% → 88.2% |
| 职业学校 | 6,300 张 | 29.5s (RTX 4090) | 48.1% → 85.6% |

**红笔清除准确率**：83.1%（可追加 500 张标注样本提升）

### 3. 法医文档检验中的颜色分离技术（综述）

| 来源 | 内容 |
|------|------|
| IEEE 2012 | [Color separation to facilitate handwriting examination](https://ieeexplore.ieee.org/document/6217832) |
| J. Forensic Sci. 2003 | [Color Separation of Signature and Stamp Inks](https://pubmed.ncbi.nlm.nih.gov/14640292/) |

**核心发现**：
- RGB 通道分离是最基础有效的手段：红墨迹在 R 通道接近白色（与纸同化），在黑墨迹在 R 通道仍为黑色
- **红外成像（IR）**：许多红墨水在红外下透明，而黑墨水吸收红外，天然分离
- 红墨水与黑墨水的交叉区域，可通过用户引导的颜色分量分离解决

**对我们的启发**：如果有条件获得红外扫描设备，红/黑分离将变得极其简单。

## 二、开源工具

### 1. IOPaint（原 Lama Cleaner）⭐⭐

| 项目 | 内容 |
|------|------|
| 仓库 | [github.com/Sanster/IOPaint](https://github.com/Sanster/IOPaint) |
| 许可证 | Apache 2.0 |

**特点**：
- 基于 LaMa 模型的图像修复工具
- 提供 WebUI，鼠标涂抹即可去除红笔
- 支持 CPU/GPU/CUDA/Apple Silicon
- 支持批量命令行处理

**安装**：
```bash
pip install iopaint
iopaint start --model=lama --device=cpu --port=8080
```

**与我们方案的对比**：
- 优势：零代码、开箱即用、修复质量高
- 劣势：无法区分红笔和红色印刷内容；需要手动涂抹（无法全自动）；LaMa 不区分笔迹颜色，会无差别修复涂抹区域

---

### 2. ABBYY FineReader — RemoveColorObjects

- 商业 OCR 引擎内置的"按颜色移除对象"功能
- 可自动检测并移除特定颜色的笔迹/标记
- 适合批量处理，但需付费

---

### 3. TextIn（合合信息）— 试卷手写笔迹去除 API

- 商业化 API，支持红/黑/蓝笔、铅笔等多种笔迹去除
- 多模态大模型驱动
- [textin.com/news/20260209103420304](https://www.textin.com/news/20260209103420304)

## 四、对下一步的建议

### 短期（Phase 4/5）：v3 生产化
- 当前 v3 对 80%+ 的红笔痕迹有效，足够开始批量清洗数据
- 先让工具跑起来，产出第一批干净数据

### 中期（Phase D 启动）：DeepErase 方案迁移
- 最值得尝试的深度学习方案，原因：
  - 弱监督训练 = 不需要大量人工标注
  - U-Net 架构简单，训练和推理都快
  - 开源代码可直接参考
- 训练数据生成：从 116 页中提取真实红笔笔画 → 叠加到干净作业上 → 自动获得标注

### 如果后续遇到 GPU 资源 + 标注预算：BiSeNetV2 + NAFA
- 目前已知的该领域最强方案
- 需要一定量的标注数据（500-1000 张）
- 但效果和生产验证都最为成熟
