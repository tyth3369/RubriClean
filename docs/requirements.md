# 需求规格说明

## 1. 项目背景

### 1.1 业务场景
- 团队正在进行数据标注实习，目标是为批改初、高中生作业的大模型构建训练数据库
- 收集到的数据是学生已完成、教师已批改并订正的作业扫描件

### 1.2 核心问题
- 扫描件中的红色笔迹（教师批改、标注、订正）会干扰模型训练
- 数据库需要的是学生原始的错误答案，而非订正后的正确答案
- 需要批量清除红笔痕迹，还原作业"未批改"的原始状态

## 2. 功能需求

### 2.1 红笔痕迹检测
| 编号 | 需求 | 优先级 |
|------|------|:----:|
| F1 | 检测标准红色笔迹（批改符号 ✓、✗、圈、分数、文字批注） | P0 |
| F2 | 检测因扫描导致的偏黑/偏暗红色笔迹 | P0 |
| F3 | 检测红色订正文字（笔画较细、颜色可能偏淡） | P1 |
| F4 | 排除印刷黑色文字和线条的干扰 | P0 |

### 2.2 红笔痕迹清除
| 编号 | 需求 | 优先级 |
|------|------|:----:|
| F5 | 使用图像修复（Inpainting）算法填充被标记区域 | P0 |
| F6 | 清除后卷面保持自然，无明显修复痕迹 | P1 |
| F7 | 不误删学生黑色/蓝色手写答案 | P0 |
| F8 | 不误删印刷题目文字和图表 | P0 |

### 2.3 批量处理
| 编号 | 需求 | 优先级 |
|------|------|:----:|
| F9 | 支持 JPG、PNG、PDF 格式输入 | P0 |
| F10 | 支持文件夹递归遍历批量处理 | P0 |
| F11 | PDF 自动拆分为单页图片处理 | P1 |
| F12 | 处理进度显示和日志记录 | P2 |

## 3. 输入输出规格

### 3.1 输入
| 属性 | 说明 |
|------|------|
| 格式 | JPG、PNG、PDF |
| 分辨率 | 典型扫描件 ~2000×3000 像素 |
| 内容 | 已批改的初高中作业（数学、物理等理科为主） |
| 笔迹颜色 | 学生：黑色/深蓝色；教师：红色 |

### 3.2 输出
| 属性 | 说明 |
|------|------|
| 格式 | 与输入同格式（或统一为 JPG） |
| 分辨率 | 保持原始分辨率 |
| 内容 | 清除红笔痕迹后的作业 |
| 命名 | 原文件名 + `_clean` 后缀（如 `hw001.jpg` → `hw001_clean.jpg`） |

## 4. 非功能性需求

### 4.1 性能
- 单张图片处理时间 < 5 秒（CPU 环境）
- 支持批量处理至少 1000 张图片不出错

### 4.2 准确性
- 标准红笔清除率 > 95%
- 学生笔迹误删率 < 2%
- 偏黑红笔清除率 > 80%

### 4.3 兼容性
- 优先支持 CPU 运行（macOS），可选 GPU 加速
- Python 3.9+ 环境

## 5. 约束条件

- 开发者算法基础较薄弱，方案需简单易懂
- 当前开发机器无 GPU，初始方案需 CPU 可运行
- 处理量大（部门级使用），需考虑稳定性

## 6. 术语表

| 术语 | 说明 |
|------|------|
| 红笔痕迹 | 教师用红色墨水笔批改作业留下的标记 |
| 偏黑红笔 | 扫描过程中因光照/设备原因红色变暗接近黑色的笔迹 |
| Inpainting | 图像修复，用周围像素填充被移除区域的技术 |
| 掩码 (Mask) | 标记需要清除的像素位置的二值图像 |
